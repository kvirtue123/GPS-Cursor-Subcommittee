# Patterns

Repeatable workflow templates in Markdown — structured guidance (not installed code) you keep in your project's `docs/` and reference in context. See the [catalog](../BEST-PRACTICES.md) and [root README](../README.md) for context.

## Where to add

- **`templates/`** — pattern template files. **Add your contributions here.**

## Contributing a pattern

Open a PR adding your `.md` file to `templates/`, or post it in the channel Feed tagged `[Pattern]`. The bar: it came from real project work, it's reusable in more than one context, and it has a clear "use this when / not when."
